# SAFE PUBLIC CLAIMS — Pulse

---

## SAFE TO CLAIM

These are accurate, verifiable statements about the product:

1. **Runs 100% in the browser with no backend** — no server, no database, no API calls for core functionality
2. **No account or sign-up required** — open the app and start tracking immediately
3. **Data is stored locally and never sent to a server** — all state lives in browser local storage
4. **Includes a library of 70 exercises with search and filter** — exercise data is part of the codebase
5. **Features a live workout session timer with automatic PR detection** — implemented and tested
6. **Tracks body weight over time with charts, a consistency heatmap, and personal records** — all three are in the Progress tab
7. **Supports kg/lb and cm/in unit switching, dark/light theme, and JSON data export** — all in Settings
8. **Built with React 19, TypeScript, Vite, and Tailwind CSS 4** — confirmed in package.json and source
9. **End-to-end tested with Playwright** — test suite exists and passes

---

## DO NOT CLAIM

Avoid these — they are unverified, exaggerated, or set expectations you can't meet:

1. **"Better than MyFitnessPal / Strava / Hevy"** — comparison claims invite scrutiny and those apps have years of development, integrations, and community behind them
2. **"Syncs across devices"** — it doesn't; data is local to one browser on one machine (unless manually exported/imported)
3. **"X users" or "Y workouts logged"** — there are no usage analytics; don't invent numbers
4. **"AI-powered coach tips"** — if tips are static/pre-written strings, don't call them AI-generated or personalized
5. **"Never loses your data"** — local storage can be cleared by the browser; don't promise permanence without warning users to export
